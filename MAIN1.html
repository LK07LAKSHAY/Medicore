📋 JSON Structure Reference

{ "patients": [ { "title": "Mr", // required — Mr|Mrs|Ms|Dr|Master|Baby "name": "JOHN DOE", // required — uppercase, max 25 chars "gender": "Male", // required — Male|Female|Other "mobile": "9876543210", // required — 10 digits "address": "123 MAIN ST", // required "age": "35", // optional (use age OR dob) "dob": "1989-04-15", // optional — YYYY-MM-DD "email": "john@email.com", // optional "healthId": "AB1234", // optional — ABHA number "weight": "72", // optional — kg "bpCategory": "Normal (120/80 mmHg)", // optional "remark": "DIABETIC", // optional "referral": { // optional "name": "Dr. Smith", "margin": 50, "type": "fixed" // fixed | percent } } ] }

ℹ️ The file must be a JSON object with a patients array. Fields marked required must be present for a row to import.

📖 Import Syntax Reference

{
  "patients": [
    {
      "title":   "Mr",
      "name":    "JOHN DOE",
      "gender":  "Male",
      "mobile":  "9876543210",
      "address": "123 MAIN STREET, CITY, PINCODE"
    }
  ]
}

These 5 fields are the minimum required to import a patient row. Any row missing one of these will be skipped with an error.

FieldRequiredValues / Format
titleRequiredMr · Mrs · Ms · Dr · Master · Baby
nameRequiredUppercase string, max 25 chars
genderRequiredMale · Female · Other
mobileRequired10-digit number (string or number)
addressRequiredUppercase string
ageOptionalNumber as string e.g. "35"
dobOptionalYYYY-MM-DD (use age OR dob)
emailOptionalValid email string
healthIdOptionalABHA / Health ID string
weightOptionalWeight in kg as string e.g. "70"
bpCategoryOptionalBP category string (see dropdown values)
remarkOptionalClinical notes, uppercase
referral.nameOptional"Self" or doctor name string
referral.marginOptionalNumber e.g. 0, 50, 100
referral.typeOptional"fixed" or "percent"
